US ATLAS hosts three shared Tier 3 computing spaces at BNL, SLAC, and UChicago,
also known as Analysis Facilities (AF). These three facilities are available to
all US ATLAS physicists and computer scientists. They are organized and managed
to support US ATLAS users' need for computing resources including login,
interactive and batch jobs, access to ATLAS data, private data storage, etc.

The AFs also support a wide variety of tools specific for analysis, including
ATLAS/CERN software in CVMFS, Grid middleware, Rucio clients, Machine Learning
packages, MPI, Jupyter Lab with PyROOT, XCache with auto data discovery, GPUs,
etc. The three facilities are backed by staff to support software environments,
unix systems and storage.

**[View the facility usage dashboard](https://atlas-kibana.mwt2.org:5601/s/analysis-facility/app/dashboards#/view/8bb58440-6145-11ed-afcf-d91dad577662?_g=%28filters%3A%21%28%29)**

The deployed documentation is available at
**[usatlas.github.io/af-docs](https://usatlas.github.io/af-docs)** and covers:

- **User Onboarding** — Applying for accounts at BNL, SLAC, and UChicago
- **Quickstart Guides** — Walkthroughs for accessing the Analysis Facilities
- **Machine Learning Containers** — Information and use of ML containers
- **ATLAS Containers** — Information and use of ATLAS containers
- **Data Storing, Accessing, and Sharing** — Using ATLAS data at AFs
- **Jupyter at Analysis Facilities** — Jupyter and how to use it at AFs
- **Data Analysis Tutorials** — Step-by-step tutorials on using AFs for analyses
- **Containers** — Container-based data processing at AFs
- **Using VSCode** — Guide on using Visual Studio Code at AFs
- **FAQ** — Answers to frequently asked questions
- **[Claude Code Skills](https://github.com/usatlas/marketplace)** — AI workflow
  assistant skills for Claude Code users

## Contributing

Corrections and improvements are welcome. To contribute:

1. [Fork this repository](https://github.com/usatlas/af-docs/fork)
2. Edit the relevant files in the `docs/` directory
3. Open a pull request against `main` with a brief description of what you
   changed

For small fixes (typos, broken links, outdated info) just open a PR directly.
For larger changes, consider opening an issue first to discuss the approach.
